  4. Re-attach from inventory
  5. Allow 30 seconds for full initialization

Common Issues

Interface Problems

HUD Not Visible or Partially Missing

Symptoms:
  • HUD attached but not visible on screen
  • Only part of HUD interface showing
  • HUD appears in wrong location
  • HUD is too small or too large
Solutions:
  1. Check HUD visibility:
    • Press Ctrl+Alt+F1 to toggle HUD visibility
    • Check viewer menu: View ‚Üí Show HUD Attachments
  2. Adjust HUD position:
    • Right-click HUD ‚Üí Edit
    • Drag to reposition on screen
    • Use position coordinates in edit window
  3. Reset HUD size:
    • Right-click HUD ‚Üí Edit
    • Check "Stretch" options in edit window
    • Reset to default size if needed
  4. Re-attach to different position:
    • Detach HUD
    • Right-click in inventory ‚Üí Attach To ‚Üí Choose different HUD position

Buttons Not Responding

Symptoms:
  • Clicking buttons produces no response
  • Some buttons work, others don't
  • Delayed response to button clicks
  • Button animations not working
Solutions:
  1. Check script status:
    • Right-click HUD ‚Üí Edit ‚Üí Contents tab
    • Verify all scripts show "Running"
    • Look for any scripts showing "Not Running"
  2. Verify click targets:
    • Ensure you're clicking directly on buttons
    • Try clicking different parts of non-responsive buttons
    • Check if transparency is too high
  3. Test with different click types:
    • Try left-click, right-click, double-click
    • Test touch and hold for 2-3 seconds
    • Use touch instead of click if available
  4. Reset HUD scripts:
    • Right-click HUD ‚Üí Edit ‚Üí Contents
    • Right-click each script ‚Üí Reset Script
    • Allow 30 seconds for reinitialization

Status and Display Issues

Incorrect Status Information

Symptoms:
  • Status shows "Disconnected" when controller is present
  • Battery/power level stuck at same value
  • Timer not updating or showing wrong time
  • Mode indicator shows wrong state
Solutions:
  1. Force status update:
    • Click the status or info button on HUD
    • Touch and hold HUD for 3 seconds
    • Ask controller to send status request
  2. Check communication link:
    • Verify you're within range of Owner HUD
    • Ensure both HUDs are using same communication channel
    • Test with simple communication command
  3. Reset status monitoring:
    • Access HUD settings menu
    • Reset monitoring systems
    • Restart status update cycles

Text Not Displaying Correctly

Symptoms:
  • Text appears garbled or corrupted
  • Missing letters or numbers
  • Text overlapping or misaligned
  • Wrong font size or style
Solutions:
  1. Reset text display:
    • Access customization settings
    • Reset font settings to defaults
    • Clear and reload text displays
  2. Check viewer settings:
    • Preferences ‚Üí Graphics ‚Üí UI Size
    • Ensure UI scaling is appropriate
    • Check font rendering options
  3. Clear texture cache:
    • Some text may be texture-based
    • Clear cache and restart viewer
    • Allow textures to reload completely

RLV Problems

RLV Not Working

RLV Commands Being Ignored

Symptoms:
  • Commands sent but no restrictions applied
  • HUD shows RLV status as red or error
  • @version command produces no response
  • Manual RLV commands don't work
Diagnostic Steps:
  1. Test RLV status:
    • Type "@version" in local chat
    • Should return RLV version information
    • If no response, RLV is not functioning
  2. Check viewer RLV settings:
    • Firestorm: Preferences ‚Üí RLVa
    • Ensure "Enable Restrained Love API" is checked
    • Check that debug messages are enabled
  3. Test simple RLV command:
    • Type "@sit=n" in local chat (prevents sitting)
    • Try to sit on something
    • Type "@sit=y" to restore sitting ability
Solutions:
  1. Enable RLV in viewer:
    • Preferences ‚Üí RLVa ‚Üí Check "Enable Restrained Love API"
    • Restart viewer after enabling
    • Test with @version command
  2. Update viewer:
    • Ensure you're using current viewer version
    • Download latest Firestorm or RLV-compatible viewer
    • Older viewers may have RLV compatibility issues
  3. Clear RLV restrictions:
    • Type "@clear" to remove all restrictions
    • Restart viewer if problems persist
    • Re-test RLV functionality

Restrictions Not Being Removed

Symptoms:
  • Restrictions remain after session end
  • Safeword doesn't clear restrictions
  • Some restrictions cleared but others remain
  • Unable to access normal avatar functions
Emergency Solutions:
  1. Clear all restrictions:
    • Type "@clear" in local chat
    • This should remove ALL active restrictions
    • Test avatar functions after clearing
  2. Check active restrictions:
    • Type "@getstatusall" in local chat
    • Lists all currently active restrictions
    • Manually clear specific restrictions if needed
  3. Disable RLV temporarily:
    • Preferences ‚Üí RLVa ‚Üí Uncheck "Enable Restrained Love API"
    • Restart viewer
    • All restrictions will be cleared
    • Re-enable RLV when ready
  4. Force restriction removal:
    • Detach all scripted objects
    • Replace outfit with clean saved outfit
    • Clear cache and restart viewer

RLV Relay Issues

Relay Not Responding

Symptoms:
  • Direct RLV works but relay commands don't
  • Controller can't send commands through relay
  • Relay shows as offline or unresponsive
  • Commands work intermittently
Solutions:
  1. Check relay attachment:
    • Ensure relay object is attached and active
    • Look for relay status indicators
    • Verify relay is receiving commands
  2. Verify relay permissions:
    • Check that controller is authorized in relay
    • Verify relay security settings
    • Ensure relay has permission to control you
  3. Test relay communication:
    • Have controller send simple test command
    • Check relay response and acknowledgment
    • Verify command reaches your HUD
  4. Reset relay system:
    • Detach and re-attach relay
    • Reset relay scripts if possible
    • Re-establish connection with controller

Connection Issues

Communication Problems

Cannot Connect to Owner HUD

Symptoms:
  • HUD shows "Searching for Controller"
  • Connection status remains "Disconnected"
  • Controller reports unable to find Wearer HUD
  • Commands sent but not received
Diagnostic Steps:
  1. Check proximity:
    • Ensure you're within 96 meters of controller
    • Both avatars should be in same region
    • No physical barriers blocking communication
  2. Verify HUD status:
    • Both HUDs should be attached and active
    • Check that scripts are running on both
    • Verify power status on both devices
  3. Test communication channels:
    • Check if HUDs are using same channel
    • Look for channel conflicts with other devices
    • Try switching to different channel
Solutions:
  1. Reset both HUDs:
    • Both controller and wearer detach HUDs
    • Wait 10 seconds
    • Re-attach both HUDs
    • Allow 30 seconds for connection establishment
  2. Manual connection:
    • Access connection settings on both HUDs
    • Manually specify target UUID
    • Force connection attempt
  3. Change communication method:
    • Switch from listen to HTTP communication
    • Try different channel ranges
    • Use backup communication protocols

Intermittent Connection Drops

Symptoms:
  • Connection works then suddenly drops
  • Frequent reconnection attempts
  • Commands sometimes work, sometimes don't
  • Status updates arrive sporadically
Solutions:
  1. Check internet stability:
    • Test internet connection stability
    • Look for network interference
    • Consider switching networks if unstable
  2. Reduce communication load:
    • Decrease status update frequency
    • Reduce number of simultaneous commands
    • Simplify communication protocols
  3. Optimize HUD performance:
    • Clear unnecessary data from memory
    • Reduce visual effects and animations
    • Close other high-resource applications

Range and Signal Issues

Connection Range Problems

Symptoms:
  • Connection lost when moving away from controller
  • Commands fail at distance
  • Signal strength warnings
  • Reduced functionality at range
Solutions:
  1. Stay within recommended range:
    • Standard range: 96 meters (SL chat range)
    • Optimal range: Within 20 meters
    • Keep line of sight when possible
  2. Use signal boosters:
    • Enable signal strength monitoring
    • Use relay objects to extend range
    • Consider HTTP-based communication for longer range
  3. Optimize for range:
    • Reduce message frequency at distance
    • Use essential communications only
    • Enable automatic reconnection

Performance Issues

Slow Response and Lag

HUD Responding Slowly

Symptoms:
  • Significant delay between button click and response
  • Status updates arrive much later than expected
  • Commands take long time to execute
  • Interface animations are choppy or slow
Solutions:
  1. Check script memory usage:
    • Right-click HUD ‚Üí Edit ‚Üí Contents
    • Check memory usage of each script
    • Look for scripts using excessive memory
  2. Reduce system load:
    • Close other attachments with heavy scripts
    • Reduce graphics settings in viewer
    • Limit other applications running on computer
  3. Optimize HUD settings:
    • Reduce update frequencies
    • Disable unnecessary visual effects
    • Use compact mode if available
  4. Clear and restart:
    • Clear viewer cache
    • Restart viewer completely
    • Re-attach HUD after restart

Memory-Related Issues

Symptoms:
  • Scripts showing "Out of Memory" errors
  • HUD functions randomly stopping
  • Error messages about memory allocation
  • Reduced functionality over time
Solutions:
  1. Identify memory-heavy scripts:
    • Check script memory usage in Contents tab
    • Look for scripts using more than 16KB
    • Identify scripts with growing memory usage
  2. Clear script memory:
    • Reset individual high-memory scripts
    • Clear log files and temporary data
    • Restart HUD to clear all memory
  3. Optimize memory usage:
    • Reduce log retention periods
    • Clear old session data
    • Disable memory-intensive features temporarily

HUD Causing Viewer Lag

  • Frame rate drops significantly
  • Avatar movement becomes jerky
  • Interface responsiveness degrades

    • Enable "Low Resource Mode"
    • Disable visual effects and animations

    • Lower viewer graphics quality
    • Reduce draw distance
    • Disable advanced lighting and shadows
