Accessing Customization Settings

1

Settings Menu

Click the "Settings" button on your HUD to access the main customization menu.

2

Configuration Categories

Navigate through different customization categories using the tabbed interface.

3

Live Preview

Most changes are applied immediately, allowing you to see the results in real-time.

4

Save and Apply

Confirm your changes and save them to your configuration profile.

Backup Tip: Before making extensive customizations, save your current configuration as a backup. This allows you to easily revert changes if needed.

Custom Color Configuration

Create your own color scheme using RGB values or HSL color picker:

# Custom Color Configuration Example
# RGB values from 0.0 to 1.0

# Primary interface color
primary_color = "0.15, 0.4, 0.9"    # Custom blue

# Secondary/accent color  
secondary_color = "0.6, 0.6, 0.6"   # Light gray

# Text color
text_color = "1.0, 1.0, 1.0"        # White

# Background color
background_color = "0.1, 0.1, 0.1"  # Dark gray

# Warning/alert color
warning_color = "1.0, 0.6, 0.0"     # Orange

# Emergency/critical color
emergency_color = "1.0, 0.2, 0.2"   # Red

# Success/confirmation color
success_color = "0.2, 0.8, 0.2"     # Green

Transparency and Opacity

Transparency Settings

  • Background Transparency: 0% (solid) to 90% (nearly invisible)
  • Button Transparency: Independent control for interactive elements
  • Text Background: Ensure text readability with background opacity
  • Status Indicators: Maintain visibility for critical information

Recommended Transparency Levels

Use Case Background Buttons Text Areas Notes

High Visibility 0-5% 0% 0-10% Maximum visibility for accessibility
Stealth Mode 70-85% 50-70% 60-80% Nearly invisible, emergency button remains visible

Custom Button Configuration

# Button Customization Example
# Position format: x, y, z (relative to HUD center)

[emergency_button]
size = "large"           # small, medium, large, extra_large
position = "0.0, 0.2, 0.0"  # Top center
color = "1.0, 0.2, 0.2"     # Red
label = "EMERGENCY"
always_visible = true

[power_button]
size = "medium"
position = "-0.15, 0.0, 0.0"  # Left center
color = "0.2, 0.8, 0.2"      # Green
label = "PWR"
show_status_light = true

[settings_button]
size = "small"
position = "0.15, -0.1, 0.0"  # Bottom right
color = "0.6, 0.6, 0.6"      # Gray
label = "SET"
quick_access = false

Screen Position Optimization

Attachment Point Recommendations

üìç Top Right (Recommended)
  • Always visible, doesn't block main view
  • Easy access for right-handed users
  • Compatible with most other HUDs
  • Good for horizontal layouts
üìç Top Left
  • Alternative for left-handed users
  • Good separation from other HUDs
  • May conflict with some UI elements
  • Suitable for vertical layouts
üìç Bottom Center
  • Excellent for wide horizontal layouts
  • Easy access, doesn't block skybox
  • May conflict with chat bar
  • Good for minimal layouts
üìç Right Center
  • Perfect for tall vertical layouts
  • Doesn't interfere with camera movement
  • Always accessible
  • Good for compact modes

Response Timing Configuration

# Behavioral Response Configuration

[response_timing]
# Command acknowledgment delay (0-5 seconds)
command_ack_delay = 0.5

# Status update frequency (60-1800 seconds)  
status_interval = 300

# Health check frequency (30-600 seconds)
health_check_interval = 120

# Emergency response time (immediate, 1-10 seconds)
emergency_response_delay = 0

# Connection timeout before auto-release (60-3600 seconds)
connection_timeout = 300

[automation_level]
# Low, Medium, High, Custom
automation_mode = "Medium"

# Auto-accept routine controller commands
auto_accept_routine = true

# Auto-report status changes
auto_report_changes = true

# Auto-optimize performance settings
auto_optimize = false

# Auto-update preferences based on usage
adaptive_learning = false

Communication Behavior

Chat and Messaging Settings

  • Chat Relay Mode:
    • All messages - Forward everything to controller
    • Filtered - Only specific types or keywords
    • Manual - Only when specifically requested
    • None - No chat forwarding
  • Message Filtering:
    • System messages (exclude/include)
    • Object messages (exclude/include)
    • Group chat (exclude/include)
    • Keyword filtering (custom lists)
  • Response Automation:
    • Auto-acknowledge commands
    • Confirm receipt of instructions
    • Report completion of tasks
    • Send periodic "still here" messages

Privacy and Discretion Settings

Privacy Feature Options Impact Recommended For
Location Sharing Always, Session only, Never Controller can track your position Trusted relationships
Online Status Visible, Hidden, Session only Shows when you're available Regular sessions
Activity Logging Full, Basic, None What information is recorded Safety and learning
Data Retention 1-90 days, Forever How long logs are kept Privacy balance

Safety and Security Behavior

Emergency Response Configuration

  • Safeword Sensitivity:
    • Exact match only
    • Case insensitive
    • Partial matches accepted
    • Multiple safeword options
  • Auto-Emergency Triggers:
    • Extended silence from controller
    • System health degradation
    • Unexpected restriction patterns
    • Connection quality issues
  • Emergency Actions:
    • Immediate restriction release
    • Notification to emergency contact
    • Auto-teleport to safe location
    • System lockdown and logging

Security Behavior Settings

# Security Configuration

[authentication]
# Controller verification method
auth_method = "uuid_and_token"  # uuid_only, uuid_and_token, enhanced
# Session token expiration (seconds)
token_expiry = 3600
# Require re-authentication after idle time
reauth_idle_time = 1800

[access_control]
# Allow multiple simultaneous controllers
multi_controller = false
# Controller priority system
priority_system = true
# Auto-reject unknown controllers
reject_unknown = true

[session_security]
# Encrypt all communications
encryption_enabled = true
# Log security events
security_logging = true
# Alert on suspicious activity
suspicious_activity_alerts = true

Notification Preferences

Notification Delivery Methods

  • HUD Display: Visual notifications on the HUD interface
  • Chat Messages: Text notifications in local chat
  • IM Messages: Direct instant messages
  • Audio Alerts: Sound notifications
  • Visual Effects: HUD color changes, flashing, or animations
  • Haptic Feedback: Controller vibration (where supported)

Notification Priority Levels

Priority Delivery Methods Persistence Override Settings
Critical All available methods Until acknowledged Cannot be disabled
High Visual + Audio + Chat 5-10 seconds Can be muted but not disabled
Medium Visual + Audio OR Chat 3-5 seconds Fully customizable
Low Visual only (default) 1-2 seconds Can be completely disabled
Info HUD display only Brief flash Optional, can be hidden

Audio Accessibility Features

Hearing Accessibility

  • Visual Sound Indicators: Screen flashes or color changes for audio alerts
  • Vibration Alerts: Controller vibration for important notifications
  • Text Descriptions: Written descriptions of audio events
  • Enhanced Visual Cues: Larger, more prominent visual notifications

Audio Sensitivity Options

  • Volume Limiting: Maximum volume caps to prevent startling
  • Gradual Alerts: Sounds that start quiet and increase gradually
  • Frequency Filtering: Avoid frequencies that may cause discomfort
  • Alert Spacing: Minimum time between audio notifications

Upload Custom Sounds

emergency_override = true     # Emergency sounds bypass all limits

Advanced Customization

Script-Level Customization

Custom Variables and Constants

Advanced users can modify script constants for deeper customization:

// Advanced Customization Constants
// Edit these in the main script for specialized behavior

// Interface timing
float STATUS_UPDATE_RATE = 1.0;        // Status refresh rate (seconds)
float BUTTON_RESPONSE_TIME = 0.1;      // Button click response delay
float ANIMATION_SPEED = 0.5;           // Interface animation speed

// Communication settings  
integer COMM_CHANNEL_BASE = 42000;     // Base communication channel
integer MAX_MESSAGE_LENGTH = 1000;     // Maximum message size
float HEARTBEAT_INTERVAL = 30.0;       // Connection heartbeat rate

// Safety parameters
float EMERGENCY_RESPONSE_TIME = 0.0;   // Emergency action delay
integer MAX_SESSION_WARNINGS = 3;      // Warnings before auto-end
float CONNECTION_GRACE_PERIOD = 60.0;  // Grace period before auto-release

// Performance tuning
integer MAX_LOG_ENTRIES = 100;         // Log history size
float MEMORY_CLEANUP_INTERVAL = 300.0; // Memory cleanup frequency
integer MAX_CONCURRENT_TASKS = 5;      // Task processing limit
