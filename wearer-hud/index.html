Overview

The Wearer HUD is the primary interface for individuals using the drone controller system. It provides essential monitoring, control, and safety features while maintaining an intuitive user experience.

Important: The Wearer HUD requires RLV/RLVa to be enabled in your viewer for full functionality. Most features will work without RLV, but some advanced controls require it.

Key Features

  • Status Monitoring: Real-time display of drone status, battery level, and connection status
  • Emergency Controls: Immediate access to safeword and emergency stop functions
  • Customization: Personalize colors, sounds, and interface preferences
  • Communication: Built-in chat and notification system
  • Session Management: Start, pause, and end drone sessions

Installation

Prerequisites

  • Second Life viewer with RLV/RLVa support (recommended: Firestorm)
  • RLV enabled in viewer preferences
  • Script permissions enabled for the drone controller system
  • Sufficient available memory for script attachment

Installation Steps

1

Download Scripts

Download the latest Wearer HUD scripts from the GitHub repository.

2

Create HUD Object

Rez a primitive object and shape it according to your preferences. The recommended size is 0.5m x 0.3m x 0.05m for optimal HUD appearance.

3

Add Scripts

Copy the following scripts into your HUD object:

  • wearer_hud_main.lsl - Main HUD functionality
  • wearer_hud_interface.lsl - User interface management
  • wearer_hud_rlv.lsl - RLV integration
  • wearer_hud_communication.lsl - Communication system
4

Configure Settings

Use the configuration notecard to set your initial preferences:

# Wearer HUD Configuration
# Version: 1.0.0

# Display Settings
hud_color = "0.2, 0.4, 0.8"
transparency = 0.1
font_size = 12

# Safety Settings
safeword = "EMERGENCY"
auto_logout_time = 3600
emergency_tp_point = "128, 128, 24"

# Communication Settings
owner_notifications = TRUE
status_broadcasts = TRUE
chat_relay = TRUE

# RLV Settings
rlv_enabled = TRUE
auto_accept_tp = FALSE
movement_restrictions = TRUE
5

Attach and Test

Attach the HUD to your screen (recommended position: Top Right) and verify all functions are working correctly.

Interface Guide

HUD Layout

The Wearer HUD features a compact, efficient design with clearly labeled sections:

Status Display

  • Power Level: Current battery/energy status (0-100%)
  • Connection: Link status with Owner HUD or Programmer Station
  • Mode: Current operational mode (Standby, Active, Maintenance)
  • Restrictions: Active RLV restrictions summary

Control Buttons

  • Power: Toggle HUD on/off
  • Emergency: Immediate safeword activation
  • Settings: Access configuration menu
  • Help: Quick help and status information

Status Indicators

Indicator Color Meaning Action Required
Power Level Green High (80-100%) None
Power Level Yellow Medium (30-79%) Consider charging soon
Power Level Red Low (0-29%) Charge immediately
Connection Green Connected None
Connection Gray Disconnected Check Owner HUD proximity
RLV Status Green Active and responding None
RLV Status Red Not enabled or error Enable RLV in viewer

Features

Emergency Systems

The Wearer HUD includes multiple layers of safety features:

Safeword System

  • Instant Access: Large, always-visible emergency button
  • Chat Command: Type your configured safeword in local chat
  • Gesture Support: Assign safeword to viewer gesture for quick access
  • Auto-Release: Automatic restriction removal on safeword use

Session Timeouts

  • Maximum Session Length: Configurable automatic logout
  • Idle Detection: Automatic pause after period of inactivity
  • Connection Loss: Auto-release if Owner HUD disconnects

Communication Features

Status Broadcasting

  • Health Monitoring: Regular status updates to Owner HUD
  • Location Sharing: Optional position reporting
  • Activity Logging: Session duration and activity tracking

Chat Integration

  • Filtered Chat: Selective message relay to owner
  • Silent Mode: Reduced notification options
  • Emergency Broadcast: Alert system for immediate attention

Customization Options

Visual Customization

  • Color Themes: Multiple pre-set color schemes
  • Transparency: Adjustable HUD opacity
  • Size Scaling: Resize HUD elements
  • Position: Flexible attachment point options

Audio Settings

  • Sound Effects: Notification sounds for different events
  • Volume Control: Individual volume levels
  • Silent Mode: Disable all audio feedback

Troubleshooting

Common Issues

HUD Not Responding

Symptoms: Buttons don't work, no status updates, HUD appears frozen

Solutions:

  • Detach and re-attach the HUD
  • Check script memory usage (should be under 64KB per script)
  • Verify all required scripts are present and running
  • Reset scripts by touching the HUD while holding Ctrl+Shift
  • Check for script errors in the viewer's script error window

Connection Issues

Symptoms: Cannot connect to Owner HUD, status shows "Disconnected"

Solutions:

  • Ensure you're within 20 meters of the Owner HUD
  • Verify both HUDs are using the same communication channel
  • Check that scripts are running on both devices
  • Try resetting both HUDs simultaneously
  • Ensure no conflicting attachments are interfering

RLV Not Working

Symptoms: Commands fail, restrictions don't apply, RLV status shows red

Solutions:

  • Enable RLV in your viewer preferences (Firestorm: Preferences → RLV)
  • Restart your viewer after enabling RLV
  • Check that RLV relay is properly configured
  • Verify the HUD has permission to control your avatar
  • Try using RLV test commands to verify functionality

Emergency Functions Not Working

Symptoms: Safeword doesn't trigger, emergency button unresponsive

Solutions:

  • Verify safeword is correctly configured in settings
  • Try using chat command: type "/1 [safeword]" in local chat
  • Use the manual detach method: Right-click HUD → Detach
  • Check for script lag affecting response times
  • Contact your owner or administrator immediately if issues persist

Performance Optimization

Reducing Memory Usage

  • Close unnecessary inventory windows
  • Reduce script-heavy attachments
  • Clear cache regularly
  • Use compact mode for reduced resource usage

Improving Response Time

  • Stay within recommended range of Owner HUD
  • Reduce graphics settings if experiencing lag
  • Use a stable internet connection
  • Avoid overcrowded areas when possible

Error Codes

Code Description Solution
E001 Script initialization failed Reset HUD scripts
E002 RLV communication error Check RLV settings in viewer
E003 Memory allocation failure Reduce script load or restart viewer
E004 Configuration file corrupted Reset to default settings
E005 Connection timeout Move closer to Owner HUD

Getting Help

If you continue to experience issues after trying these solutions:

  • Check the FAQ section for additional solutions
  • Review the RLV Reference for RLV-specific issues
  • Contact your session owner or system administrator
  • Visit our GitHub Issues page to report bugs
Safety Reminder: If you experience any safety concerns or cannot access emergency functions, immediately detach the HUD manually by right-clicking it and selecting "Detach". Your safety is the highest priority.
