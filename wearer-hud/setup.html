Wearer HUD Setup Guide

Complete step-by-step instructions for installing, configuring, and testing your Wearer HUD for the first time.

Prerequisites

Important: Please read through this entire setup guide before beginning installation. Understanding the process and safety requirements is essential for proper operation.

System Requirements

Component Requirement Notes
Second Life Viewer Current version (recommended: Firestorm) Must support RLV/RLVa
RLV Status Enabled and functional Required for full functionality
Script Memory Minimum 128KB available For all HUD scripts combined
Attachment Points 1 free HUD attachment point Recommended: Top Right or Bottom
Permissions Script permissions enabled Debit, Attach, Animation, RLV

Knowledge Requirements

  • Basic SL Skills: Comfortable with inventory, attachments, and basic scripting concepts
  • RLV Understanding: Familiar with RLV/RLVa and its safety implications
  • Safety Awareness: Understanding of safewords and emergency procedures
  • Communication: Ability to communicate clearly with potential controllers

Safety Preparation

Before You Begin: The Wearer HUD is designed with multiple safety systems, but you should always ensure you have a trusted controller and understand all safety procedures before use.

Essential Safety Steps

  1. Choose Your Safeword: Select a memorable word that will immediately stop all restrictions
  2. Set Emergency Contact: Have backup communication method with trusted friend
  3. Know Your Limits: Decide on maximum session duration and restriction levels
  4. Test Environment: Use private area for initial testing
  5. Trusted Controller: Only work with experienced, trusted operators

Download & Preparation

Getting the Scripts

1

Access the Repository

Visit the official script repository: https://github.com/BrittRose-SL/drone-controller-scripts

2

Download Latest Release

Click on "Releases" and download the latest stable version. Look for files marked "wearer-hud" in the release assets.

3

Extract Files

Extract the downloaded archive. You should have the following Wearer HUD files:

  • wearer_hud_main.lsl - Core HUD functionality
  • wearer_hud_interface.lsl - User interface management
  • wearer_hud_rlv.lsl - RLV integration handler
  • wearer_hud_communication.lsl - Communication system
  • wearer_hud_safety.lsl - Emergency and safety systems
  • wearer_hud_config.txt - Configuration notecard template

Preparing Your Inventory

  1. Create Folder: Make a new folder in your inventory called "Drone Controller - Wearer HUD"
  2. Import Scripts: Copy all LSL files into Second Life using your preferred method
  3. Organize Assets: Place all scripts and configuration files in your new folder
  4. Backup: Make a backup copy of all files before modification
Tip: You can import LSL files by copying the text and pasting into new scripts in SL, or by using import tools if your viewer supports them.

Installation Process

Creating the HUD Object

1

Rez a Primitive

In a sandbox or area where you have build rights:

  • Right-click on the ground and select "Create"
  • Create a cube primitive
  • Resize it to approximately 0.5m x 0.3m x 0.05m for optimal HUD appearance
2

Shape and Texture the HUD

Customize the appearance:

  • Set transparency to about 10-20% for visibility without obstruction
  • Choose a color scheme that works with your preferences
  • Add any desired textures or materials
  • Consider adding text or symbols for button identification
3

Prepare for Scripts

Set up the object for scripting:

  • Right-click the object and select "Edit"
  • Go to the "Contents" tab
  • Ensure "Running Scripts" is checked
  • Clear any default scripts if present

Installing Scripts

1

Add Main Script First

Drag wearer_hud_main.lsl from your inventory into the object's Contents tab. This script manages communication between all other components.

2

Add Support Scripts

Add the remaining scripts in this order:

  1. wearer_hud_safety.lsl (highest priority)
  2. wearer_hud_rlv.lsl
  3. wearer_hud_communication.lsl
  4. wearer_hud_interface.lsl (last)
3

Add Configuration

Create a new notecard called "config" and paste the contents of wearer_hud_config.txt into it, then drag it into the object's Contents.

Script Compilation and Verification

  1. Check Compilation: Ensure all scripts compile successfully without errors
  2. Monitor Memory: Total memory usage should be under 128KB
  3. Test Communication: Scripts should establish inter-communication automatically
  4. Watch Chat: Look for initialization messages in local chat
Troubleshooting: If any scripts show compilation errors, double-check that you copied the complete script code. Missing functions or syntax errors will prevent proper operation.

Initial Configuration

Basic Configuration Settings

Edit the "config" notecard to customize your HUD. Here's the basic configuration structure:

# Wearer HUD Configuration
# Version: 1.0.0
# User: [Your Avatar Name]
# Date: [Current Date]

# === SAFETY SETTINGS ===
# Primary safeword - NEVER forget this!
safeword = "EMERGENCY"

# Maximum session duration in seconds (7200 = 2 hours)
max_session_time = 7200

# Auto-logout if no owner communication (seconds)
connection_timeout = 300

# Emergency teleport coordinates (your safe location)
emergency_tp_point = "128, 128, 24"

# === DISPLAY SETTINGS ===
# HUD color (RGB values 0.0 to 1.0)
hud_color = "0.2, 0.4, 0.8"

# Background transparency (0.0 = invisible, 1.0 = opaque)
transparency = 0.1

# Font size for text elements
font_size = 12

# Compact mode for smaller screens
compact_mode = FALSE

# === COMMUNICATION SETTINGS ===
# Send status updates to owner
owner_notifications = TRUE

# Frequency of status updates (seconds)
status_interval = 300

# Relay local chat to owner
chat_relay = TRUE

# Share location information
location_sharing = TRUE

# === RLV SETTINGS ===
# Enable RLV functionality
rlv_enabled = TRUE

# Auto-accept teleport requests from owner
auto_accept_tp = FALSE

# Enable movement restrictions
movement_restrictions = TRUE

# Enable attachment controls
attachment_controls = TRUE

# === AUDIO SETTINGS ===
# Enable sound effects
sound_enabled = TRUE

# Sound volume (0.0 to 1.0)
sound_volume = 0.5

# Notification sounds
notification_sounds = TRUE

# === ADVANCED SETTINGS ===
# Debug mode (shows additional information)
debug_mode = FALSE

# Log all activities
activity_logging = TRUE

# Encryption key for secure communication (leave blank for auto-generation)
encryption_key = ""

Customizing Your Settings

Safety Configuration

  • Safeword: Choose something memorable but not commonly used in conversation
  • Session Time: Start with shorter durations (30-60 minutes) until experienced
  • Emergency TP: Set to a private, safe location you own or have access to
  • Connection Timeout: Recommended 5-10 minutes for safety

Interface Customization

  • Colors: Choose colors that are visible but not distracting
  • Transparency: Balance visibility with screen space
  • Compact Mode: Use for smaller screens or minimal interface
  • Font Size: Adjust for readability at your screen resolution

Communication Preferences

  • Status Updates: Enable for safety monitoring
  • Chat Relay: Consider privacy implications
  • Location Sharing: Required for some features
  • Notifications: Balance information with distraction
Security Note: The encryption key field can be left blank for automatic generation, or you can specify a custom key for enhanced security with specific controllers.

Testing & Verification

Pre-Attachment Testing

1

Script Status Check

Before attaching, verify all systems are operational:

  • Touch the HUD object to trigger initialization
  • Check local chat for status messages
  • Verify all scripts show "Running" status
  • Look for any error messages or warnings
2

Configuration Validation

Confirm settings are loaded correctly:

  • Touch and hold for 3 seconds to access debug info
  • Verify safeword is set correctly
  • Check that emergency coordinates are valid
  • Confirm RLV status is detected properly

Attachment and Interface Testing

1

Attach the HUD

Right-click the object and select "Attach To" → "Attach to HUD" → Choose your preferred position (Top Right recommended).

2

Interface Verification

Test all interface elements:

  • Power Button: Should toggle HUD on/off
  • Emergency Button: Should be clearly visible and responsive
  • Status Display: Should show current status information
  • Settings Access: Should open configuration menu
3

RLV Integration Test

Verify RLV functionality (if enabled):

  • HUD should detect RLV status automatically
  • RLV indicator should show green if working
  • Test basic RLV command response
  • Verify safeword triggers RLV release

Safety System Testing

Emergency Functions Test

  1. Safeword Test: Type your safeword in local chat - HUD should respond immediately
  2. Emergency Button: Click the emergency button - should trigger immediate safety response
  3. Manual Detach: Practice detaching the HUD manually if needed
  4. Timeout Test: Verify connection timeout works if owner disconnects

Communication Testing

  1. Status Updates: Verify regular status messages are generated
  2. Chat Relay: Test if local chat is properly relayed (if enabled)
  3. Location Updates: Confirm position information is accurate
  4. System Messages: Check that important events generate notifications
Testing Complete: If all tests pass successfully, your Wearer HUD is ready for use! Save the configured object to your inventory for future use.

First Use Guide

Before Your First Session

Final Preparation Checklist

  • ✅ HUD tested and functioning properly
  • ✅ Safeword memorized and tested
  • ✅ Emergency procedures understood
  • ✅ Session limits decided and configured
  • ✅ Trusted controller identified and briefed
  • ✅ Private, safe environment secured
  • ✅ Backup communication method established

Starting Your First Session

1

Pre-Session Communication

Before beginning, communicate with your controller:

  • Confirm safeword and emergency procedures
  • Discuss session goals and limits
  • Verify both HUDs are operational
  • Agree on session duration
2

HUD Activation

Activate your Wearer HUD:

  • Click the power button to turn on the HUD
  • Verify connection with Owner HUD
  • Confirm all status indicators are green
  • Check that emergency functions are accessible
3

Session Initiation

When ready to begin:

  • Give final consent to your controller
  • Watch for initial commands and restrictions
  • Verify that restrictions are applied correctly
  • Maintain communication as requested

During Your Session

Monitoring Your Status

  • Status Display: Keep an eye on battery level and connection status
  • Health Indicators: Watch for any warning signs or errors
  • Communication: Maintain regular check-ins as agreed
  • Comfort Level: Monitor your physical and emotional comfort

What to Expect

  • Gradual Progression: Good controllers will start slowly and build up
  • Regular Check-ins: Expect frequent status questions, especially initially
  • Clear Communication: Commands should be clear and restrictions explained
  • Respect for Limits: Your stated limits should be honored

Ending Your Session

Normal Session End

  1. Controller initiates session termination sequence
  2. All restrictions are gradually removed
  3. HUD returns to standby mode
  4. Final status check and debrief
  5. HUD can be powered down or detached

Emergency Session End

  1. Use safeword immediately if needed
  2. Click emergency button if safeword fails
  3. Manually detach HUD if systems fail
  4. Contact emergency support if needed
  5. Document any issues for future safety
Remember: Your safety and wellbeing are the highest priority. Never hesitate to use your safeword or emergency systems if you feel uncomfortable or unsafe.

Post-Session Care

Immediate Aftercare

  • Status Check: Ensure you feel physically and emotionally well
  • Hydration: Drink water and take care of basic needs
  • Decompression: Take time to process the experience
  • Communication: Discuss the session with your controller

System Maintenance

  • HUD Status: Verify HUD returned to normal operation
  • Settings Review: Update any preferences based on experience
  • Log Review: Check activity logs for any issues
  • Backup: Save current HUD configuration
