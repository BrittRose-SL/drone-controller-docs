üìÑ Configuration File Management

The _config_aria.cfg notecard is the primary configuration file for your A.R.I.A. system. It uses a simple key=value format for easy editing.

Configuration File Format

_config_aria.cfg 
# A.R.I.A. Controller Configuration
# Lines starting with # are comments
# Format: KEY=VALUE (no spaces around =)

# === BASIC SYSTEM SETTINGS ===
UNIT_NAME=A.R.I.A. Unit
HOME_LANDMARK=
WEARER_ADMIN_MODE=0

# === USER MANAGEMENT ===
# UUIDs separated by commas (no spaces)
ADMINISTRATORS=12345678-1234-1234-1234-123456789012
TRUSTED_USERS=87654321-4321-4321-4321-210987654321,11111111-2222-3333-4444-555555555555

# === POWER MANAGEMENT ===
BATTERY_DRAIN_RATE=1.0
LOW_BATTERY_THRESHOLD=25.0
CRITICAL_BATTERY_THRESHOLD=5.0
AUTO_SHUTDOWN_ENABLED=1

# === COMMUNICATION ===
HUD_CHANNEL=-18795462
STATION_CHANNEL=-18795462
VERBOSE_LOGGING=0

# === MODULE DEFAULTS ===
DEFAULT_RELAY_MODE=0
AUTO_APPLY_RESTRICTIONS=0
BATTERY_BASED_RESTRICTIONS=1

# === PERFORMANCE TUNING ===
TIMER_INTERVAL=60.0
MEMORY_CLEANUP_INTERVAL=300.0
MAX_HISTORY_ENTRIES=10

Editing the Configuration File

  1. Open the Controller's Inventory

    Right-click your A.R.I.A. Controller ‚Üí Edit ‚Üí Content tab

  2. Find the Configuration Notecard

    Look for _config_aria.cfg in the contents list

  3. Edit the Notecard

    Double-click the notecard to open it in the text editor

    Important: Use the exact format shown above. No spaces around the = sign, and UUIDs must be valid.
  4. Save and Close

    Save the notecard and close the editor. The system will automatically reload the configuration.

  5. Verify Changes

    Touch the controller and check STATUS to confirm your changes were applied.

üë• User Lists & Permissions

Configure who can access and control your A.R.I.A. system through user lists and permission settings.

User Categories

üî¥ Administrators

Full system control including power management and user list modification.

ADMINISTRATORS=uuid1,uuid2,uuid3
  • Can power system on/off
  • Can modify user lists
  • Access to all modules and settings
  • Can access diagnostic functions
  • Emergency override capabilities

üü° Trusted Users

Access to RLV modules and configuration viewing without system control.

TRUSTED_USERS=uuid1,uuid2,uuid3
  • Can control RLV modules
  • Can view system status
  • Cannot modify power state
  • Cannot change user lists
  • Limited diagnostic access

üü¢ Wearer

The person wearing the controller with configurable access levels.

WEARER_ADMIN_MODE=0 or 1
  • Basic status access always available
  • SOS function always accessible
  • HOME teleport (if configured)
  • Admin mode: Full privileges when enabled
  • Can always detach the controller

Advanced User Management

Wearer Admin Mode Configuration

New in v10.3: Configure whether the wearer has administrative privileges.

WEARER_ADMIN_MODE=0

Standard mode: Wearer has basic access only

Recommended for shared control
WEARER_ADMIN_MODE=1

Admin mode: Wearer has full administrative privileges

Recommended for self-control
Security Consideration: Wearer admin mode allows the wearer to modify user lists, power settings, and all system configurations. Enable only when appropriate.

Finding User UUIDs

Method 1: Profile Inspection

  1. Right-click the person ‚Üí Profile
  2. In the profile window, look at the URL or inspect element
  3. Copy the UUID from the profile data

Method 2: LSL Script Helper

Use a simple LSL script to get UUIDs:

llOwnerSay("My UUID: " + (string)llGetOwner());

Method 3: In-Menu Addition

Use the Permissions module menu to add users by name or proximity.

üîß Diagnostics Module

DEFAULT_RELAY_MODE

Default RLV relay mode on startup

0=Off (Secure) 1=Ask 2=Trusted 3=Everyone
Security: Default is 0 (Off) for maximum security. Only change if you understand the implications.

üéÆ RLV Modules General

AUTO_APPLY_RESTRICTIONS

Automatically apply restrictions on attachment

AUTO_APPLY_RESTRICTIONS=0 or 1
  • 0: Start with no restrictions (recommended)
  • 1: Apply default restrictions immediately
BATTERY_BASED_RESTRICTIONS

Enable automatic restrictions based on battery level

BATTERY_BASED_RESTRICTIONS=1
  • 0: Disable battery-based restrictions
  • 1: Enable progressive restrictions (recommended)

üîã Power Management Tuning

Configure battery simulation behavior, drainage rates, and power-based restrictions.

‚ö° Battery Drain Configuration

BATTERY_DRAIN_RATE

Base battery drain per minute

BATTERY_DRAIN_RATE=1.0
  • 0.5: Slow drain (200 minutes = ~3.3 hours)
  • 1.0: Standard drain (100 minutes = ~1.7 hours)
  • 2.0: Fast drain (50 minutes)
  • 5.0: Very fast drain (20 minutes)

‚ö†Ô∏è Battery Thresholds

LOW_BATTERY_THRESHOLD

Percentage when low battery warnings begin

LOW_BATTERY_THRESHOLD=25.0
CRITICAL_BATTERY_THRESHOLD

Percentage when critical restrictions apply

CRITICAL_BATTERY_THRESHOLD=5.0
AUTO_SHUTDOWN_ENABLED

Enable automatic shutdown at 0% battery

AUTO_SHUTDOWN_ENABLED=1

üîÑ Power-Based Restriction Levels

The system applies progressive restrictions based on battery level:

100-50% Battery

Full functionality available

49-25% Battery

Minor restrictions, low battery warnings

24-5% Battery

Significant restrictions, power saving mode

Below 5% Battery

Emergency restrictions, essential functions only

‚ö° Performance Optimization

Configure system performance parameters to optimize script execution and memory usage.

‚è±Ô∏è Timing Configuration

TIMER_INTERVAL

Main system timer interval in seconds

TIMER_INTERVAL=60.0
30.0 = More responsive, higher script time 60.0 = Balanced (recommended) 120.0 = Less responsive, lower script time
MEMORY_CLEANUP_INTERVAL

Memory cleanup and garbage collection interval

MEMORY_CLEANUP_INTERVAL=300.0
  • 300.0 = 5 minutes (recommended)
  • Clears diagnostic history
  • Optimizes memory usage
  • Reduces script lag over time

üíæ Memory Management

MAX_HISTORY_ENTRIES

Maximum diagnostic history entries to retain

MAX_HISTORY_ENTRIES=10
  • 5: Minimal memory usage
  • 10: Balanced (recommended)
  • 20: Extended history, higher memory usage
  • 50: Maximum history, may impact performance

üíæ Backup & Restore

Protect your configuration and ensure easy recovery with proper backup and restore procedures.

üìã Manual Configuration Backup

  1. Copy Configuration Notecard

    Right-click controller ‚Üí Edit ‚Üí Content ‚Üí Copy _config_aria.cfg to inventory

  2. Rename with Date

    Rename copied notecard to include date: _config_aria_backup_2025-09-05.cfg

  3. Store Safely

    Keep backup notecards in a dedicated inventory folder

ü§ñ Complete System Backup

  1. Take Controller into Inventory

    Detach controller and copy the entire object to create a backup

  2. Create Backup Copy

    Right-click controller in inventory ‚Üí Copy ‚Üí Paste

  3. Rename Backup

    Name backup with date: A.R.I.A. Controller BACKUP 2025-09-05

  4. Store in Safe Location

    Keep backups in a dedicated folder with clear organization

üîÑ Restore Procedures

Configuration Restore

  1. Locate your backup configuration notecard
  2. Copy the backup notecard to the controller's content
  3. Rename it to _config_aria.cfg (replacing the current one)
  4. The system will automatically reload the configuration
  5. Verify settings through STATUS menu

Complete System Restore

  1. Detach current controller (if attached)
  2. Delete current controller (if needed)
  3. Rez backup controller from inventory
  4. Verify all modules are functioning
  5. Attach and test full functionality

üìÖ Recommended Backup Schedule

Before Major Changes Always backup before modifying user lists or system settings
Weekly Regular configuration backups if actively customizing
Before Updates Complete system backup before installing new versions
Monthly Archive old backups and create fresh complete system backup

üõ†Ô∏è Configuration Troubleshooting

Common configuration issues and their solutions.

‚ùå Configuration Not Loading

Symptoms:

  • Default settings despite custom notecard
  • "Configuration not found" messages
  • Changes not taking effect

Solutions:

  • Verify notecard name is exactly _config_aria.cfg
  • Check notecard is in the root prim contents
  • Ensure notecard format follows key=value pattern
  • Look for syntax errors in the notecard
  • Reset the main module script if needed

‚ö†Ô∏è Invalid User UUIDs

Symptoms:

  • Users not recognized despite being in list
  • "Invalid UUID" warnings
  • Permission errors for configured users

Solutions:

  • Verify UUIDs are in correct format (8-4-4-4-12 characters)
  • Check for extra spaces or characters
  • Use valid UUIDs only (not avatar names)
  • Separate multiple UUIDs with commas only
  • Test UUIDs in a simple script first

üîã Power Management Issues

Symptoms:

  • Battery draining too fast/slow
  • Restrictions not applying at thresholds
  • System not shutting down at 0%

Solutions:

  • Check BATTERY_DRAIN_RATE is reasonable (0.1-10.0)
  • Verify threshold values are logical (0-100)
  • Ensure BATTERY_BASED_RESTRICTIONS=1 if desired
  • Check AUTO_SHUTDOWN_ENABLED setting
  • Review module count impact on drain

üì° Communication Problems

Symptoms:

  • HUDs not finding A.R.I.A. unit
  • Commands not working from HUDs
  • Intermittent connectivity issues

Solutions:

  • Verify all components use same channel
  • Check channel is negative number
  • Ensure components are within range (96m)
  • Look for channel conflicts with other objects
  • Reset all components if needed
