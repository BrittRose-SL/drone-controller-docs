üöÄ First-Time Setup

Step 1: Initial Configuration

  1. Rez the Controller

    Drag the A.R.I.A. Controller from inventory to the ground near you.

    Important: Do NOT attach the controller yet. Configure it first while it's rezzed on the ground.
  2. Check System Status

    Touch the controller to access the main menu. You should see:

    === A.R.I.A. CONTROL SYSTEM ===
    Unit: A.R.I.A. Unit
    Power: ON | Battery: 100%
    Modules: 10
  3. Verify Module Loading

    The controller will automatically initialize all modules. Watch for these messages in local chat:

    • "A.R.I.A. Main Module online"
    • "Module registered: [Module Name]" (√ó10 modules)
    • "Configuration loaded"

Step 2: Basic Configuration

  1. Set Administrator

    By default, you (the owner) are set as the primary administrator. To add additional administrators:

    • Touch controller ‚Üí MODULES ‚Üí Permissions
    • Select "Add Admin" and enter the user's UUID or name
    • Confirm the addition
  2. Configure Unit Name

    Personalize your controller with a custom name:

    • Touch controller ‚Üí MODULES ‚Üí Permissions
    • Select "Unit Name"
    • Enter your desired name (e.g., "Sarah's A.R.I.A.")
  3. Test Basic Functions

    Verify core functionality before attaching:

    • Touch controller ‚Üí STATUS (check all systems)
    • Touch controller ‚Üí MODULES (verify all 10 modules listed)
    • Try opening individual module menus

üë§ Attaching the Controller

Attachment Process

  1. Prepare for Attachment

    Before attaching, ensure:

    • RLV is enabled in your viewer
    • You understand the controller's capabilities
    • Trusted users are properly configured
    • You have a safe word/removal method planned
  2. Attach the Controller

    Right-click the controller object and select:

    Attach ‚Üí Neck
    or your preferred attachment point
  3. Initial Attachment Messages

    Upon attachment, you should see initialization messages:

    A.R.I.A. Controller attached and initializing...
    Battery: 100% | Power: ON
    All modules online
    A.R.I.A. system ready
  4. Position Adjustment

    If needed, adjust the controller position:

    • Right-click controller ‚Üí Edit
    • Use position and rotation tools
    • Ensure comfortable fit and appearance
    • Stop editing when satisfied
Congratulations! Your A.R.I.A. Controller is now active and ready for use.

üéÆ First Use Guide

Understanding Your Controller

Now that your A.R.I.A. Controller is attached, here's what you need to know:

üîã Power Management

Battery starts at 100% and drains over time based on usage. Monitor your power level and recharge as needed.

üõ°Ô∏è Security Features

Access is controlled by administrator/trusted user lists. Only authorized users can modify settings.

üì° Communication

The controller communicates with HUDs and other devices on channel -18795462.

üÜò Emergency Features

SOS function available to alert administrators. Controller can always be detached manually.

Menu ‚Üí MODULES lists all available RLV modules. Click any module to access its specific controls.

üõ°Ô∏è Safety & Emergency Procedures

Important Safety Information: Please read this section carefully before using advanced features.

Emergency Removal

üÜò Immediate Detachment

Right-click controller ‚Üí Detach - This works even if RLV restrictions are active.

üìû SOS Function

Touch controller ‚Üí SOS - Alerts all administrators immediately.

‚ö° Emergency Power Off

Say "@clear" in local chat - Clears most RLV restrictions (if permitted).

üîÑ Viewer Restart

Restart your viewer - Clears all temporary RLV restrictions.

Safe Use Guidelines

  • Always communicate boundaries and limits clearly
  • Start with minimal restrictions and gradually increase
  • Ensure trusted users understand the system capabilities
  • Regularly check battery levels to avoid unexpected shutdowns
  • Keep backup communication methods available
  • Never use the system with unknown or untrusted individuals
  • Understand that you can always detach the controller

üõ†Ô∏è Quick Troubleshooting

üö´ Controller Won't Attach

Solutions:

  • Check attachment point isn't already occupied
  • Verify object permissions allow wearing
  • Try attaching to a different attachment point
  • Ensure you're the owner of the object

‚ö†Ô∏è RLV Not Working

Solutions:

  • Enable RLV in viewer preferences
  • Restart viewer after enabling RLV
  • Check for RLV menu in Avatar menu
  • Verify you're using an RLV-compatible viewer

üîã Battery Draining Fast

Solutions:

  • Reduce number of active modules
  • Check for script errors in modules
  • Use power-off when not needed
  • Update to latest software version

üìû Can't Access Menu

Solutions:

  • Ensure you have proper access permissions
  • Check if controller is powered on
  • Try detaching and re-attaching
  • Reset scripts if necessary
